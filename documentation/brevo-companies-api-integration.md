# Brevo Companies API Integration

## Overview

This document describes how the Brevo Companies API integration works in the Little Microphones system. Companies represent schools/organizations, and contacts (educators) can be linked to them.

## Key Concepts

### Company ID vs School Place ID

- **Brevo Company ID**: Automatically generated by Brevo when creating a company (e.g., "123e4567-e89b-12d3-a456-426614174000")
- **School Place ID**: Google Places ID from Memberstack (e.g., "ChIJN1t_tDeuEmsRUsoyG83frY4")
- We store `SCHOOL_PLACE_ID` as a company attribute to maintain the link between our system and Brevo

### API Endpoints Used

1. **GET /companies** - List all companies with pagination
2. **POST /companies** - Create a new company
3. **GET /companies/{id}** - Get specific company by Brevo ID
4. **PATCH /companies/{id}** - Update company attributes
5. **PATCH /companies/link-unlink/{id}** - Link/unlink contacts to company

## Implementation Details

### Creating/Updating Companies

When a user has `school-place-id` in their Memberstack data:

1. **Check if company exists**: Search all companies for matching `SCHOOL_PLACE_ID` attribute
2. **Create or Update**: 
   - If not found: Create new company with school data
   - If found: Update existing company with latest data
3. **Store Brevo ID**: Return the Brevo-generated company ID for linking

```javascript
const companyResult = await createOrUpdateBrevoCompany(schoolData);
// Returns: { success: true, companyId: "brevo-id", schoolPlaceId: "google-places-id" }
```

### Linking Contacts to Companies

After creating/updating a company:

1. **Get Contact ID**: Retrieve Brevo contact ID from email
2. **Link Contact**: Use `/companies/link-unlink/{id}` endpoint
3. **Update Attributes**: Add role-specific data to contact

```javascript
const linkResult = await linkContactToCompany(email, companyResult.companyId, {
  EDUCATOR_ROLE: "Teacher",
  IS_PRIMARY_CONTACT: false
});
```

### Company Attributes

Companies can have these custom attributes (must be created in Brevo dashboard first):

- `SCHOOL_PLACE_ID` - Google Places ID (critical for identification)
- `SCHOOL_NAME` - School name
- `SCHOOL_WEBSITE` - Website URL
- `SCHOOL_PHONE` - Phone number
- `SCHOOL_ADDRESS` - Full address
- `SCHOOL_CITY`, `SCHOOL_STATE_PROVINCE`, `SCHOOL_COUNTRY`
- `SCHOOL_LATITUDE`, `SCHOOL_LONGITUDE` - For mapping
- `TOTAL_STUDENTS`, `TOTAL_EDUCATORS`, `TOTAL_CLASSES` - Metrics

## Helper Functions

### `getBrevoCompanyBySchoolId(schoolPlaceId)`
Searches all companies to find one with matching `SCHOOL_PLACE_ID` attribute. Handles pagination automatically.

### `getBrevoCompany(companyId)`
Gets company by Brevo ID directly.

### `getCompanyContacts(companyId)`
Returns all contacts linked to a specific company.

### `updateCompanyMetrics(companyId, metrics)`
Updates company metrics like total students, educators, etc.

## Field Clearing Issue

To clear fields in Brevo, send `null` values instead of omitting fields:
- Empty field in Memberstack → `null` sent to Brevo
- This should clear the field (pending verification)

## Troubleshooting

1. **Company not created**: Check if `SCHOOL_PLACE_ID` exists in member data
2. **Link fails**: Ensure both contact and company exist in Brevo first
3. **404 errors**: Using wrong endpoint (e.g., `/crm/companies` instead of `/companies`)
4. **Attributes not updating**: Ensure attributes are created in Brevo dashboard first
